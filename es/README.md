# Unit Testing 101

## Lección 1: Escribe tus primeros units tests

En la lección 1, aprenderás cómo escribir algunos unit tests para Stringie, una librería para transformar cadenas. Los desarrolladores de Stringie escucharon acerca del unit testing y escribieron algo parecido a un unit test. 

En esta lección, aprenderás: a escribir dos o tres unit tests para uno de los métodos de Stringie, a cómo organizar tus tests en una solución y a cómo nombrar tus proyectos de testing, clases y métodos. 

**[Empieza la Lección 1](Leccion1/README.md)**.

## Lección 2: Corrige algunos tests

En la lección 2, tendrás que corregir algunos unit tests. Esta vez, los desarrolladores de Stringie cometieron ciertos errores mientras escribían algunos unit tests, olvidaron usar una convención de nombres, entre otras cosas más. 

Así que, en esta lección deberás arreglar los test escritos erróneamente por los desarrolladores de Stringie. 

**[Empieza la Lección 2](Leccion2/README.md)**.

## Lección 3: Renombra tus tests

En la lección 3, tendrás que renombrar algunos unit tests. En esta ocasión, los desarrolladores de Stringie corrigieron casi todos los errores en sus tests, excepto uno: los nombres de los tests. 

Aquí, deberás seguir una convención de nombres, para corregir este error.

**[Empieza la Lección 3](Leccion3/README.md)**.

## Créditos

Stringie fue creada sólo con propósitos de aprendizaje. Los métodos de Stringie fueron tomados de [FluentStrings](https://github.com/MSayfullin/FluentStrings).
